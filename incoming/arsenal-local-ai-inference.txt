## Executive Summary
In offensive security, data privacy is paramount. Transmitting proprietary binaries, zero-day research, or sensitive client logs to public AI endpoints is an unacceptable operational risk. This architecture creates an isolated Controller/Agent ecosystem for private, local inference.

---

## 1. Threat Model & Design
* **Data Locality:** All inference requests remain within the local network boundary.
* **Ephemeral Processing:** Data is piped via `stdin`, ensuring zero persistence on the inference server's storage.
* **Resource Management:** By offloading heavy compute to a dedicated host, we avoid resource starvation on the primary penetration testing rig.

---

3. Infrastructure: The Controller Bridge
The bridge allows the Parrot OS terminal to act as a client. It intercepts stdin, allowing for seamless integration with standard tools like nmap or grep.

Repo Reference: jericho/ai-inference/scripts/parrot_agent.py

Python
#!/usr/bin/env python3
import requests, sys, json

OLLAMA_HOST = "[http://192.168.1.50:11434/api/generate](http://192.168.1.50:11434/api/generate)"

def stream_response(data, model="deepseek-coder"):
    try:
        response = requests.post(
            OLLAMA_HOST,
            json={"model": model, "prompt": f"Analyze: {data}", "stream": True},
            stream=True, timeout=60
        )
        response.raise_for_status()
        for line in response.iter_lines():
            if line:
                chunk = json.loads(line).get("response", "")
                sys.stdout.write(chunk); sys.stdout.flush()
    except Exception as e:
        print(f"\n[-] Bridge Error: {e}", file=sys.stderr)
        sys.exit(1)

if __name__ == "__main__":
    stream_response(sys.stdin.read())
