Modern offensive research relies on AI co-pilots, but transmitting zero-day context to third-party endpoints is a fatal OpSec failure. This architecture creates an isolated Controller/Agent ecosystem for private, local inference.

### I. Architectural Debt & Security Hardening
* Network Isolation: The Inference Server acts as a black box. Communication is restricted to the internal network.
* Ephemeral Context: The Python bridge utilizes stdin pipes to ensure no sensitive target data is persisted to the Inference Server's disk.
* Service Persistence: The Windows node leverages background execution to prevent session timeouts.

III. Implementation: Controller Bridge (Parrot OS)
The bridge implementation requires robust error handling to prevent "broken pipes" during long-running analysis.

Repo: jericho/ai-inference/scripts/parrot_agent.py

Python
#!/usr/bin/env python3
import requests, sys, json
OLLAMA_HOST = "[http://192.168.1.50:11434/api/generate](http://192.168.1.50:11434/api/generate)"

def stream_response(prompt, model):
    try:
        response = requests.post(OLLAMA_HOST, json={"model": model, "prompt": prompt, "stream": True}, stream=True, timeout=60)
        response.raise_for_status()
        for line in response.iter_lines():
            if line:
                chunk = json.loads(line).get("response", "")
                sys.stdout.write(chunk); sys.stdout.flush()
    except requests.exceptions.Timeout:
        print("\n[-] Error: Inference Server timed out")
        sys.exit(1)

if __name__ == "__main__":
    stream_response(sys.stdin.read(), "deepseek-coder")
